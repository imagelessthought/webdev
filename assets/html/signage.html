Universal Digital Signage

Centrally managed digital sign and screensaver content delivery

Description:

The solution created will allow digital signs, kiosks, and screensavers to be pointed to a webserver where the content is stored. Content is provided through html playlists: each playlist may have a unique design. Each device that is registered should be able to retrieve a customized playlist (checks by hostname) or use a playlist created for a channel or particular group. Signage admins should be able to create, edit, and delete playlists (unique and channels), add\remove images, and assign playlists to devices through an admin portal.

Components

Webserver (Hosts Content):

  • Upload images to Image Library

    • The user should be able to upload new images to the image library through the admin portal. Users should be able to create separate libaries. Libraries will be stored in \content\images\library-name. The user should be able to move and copy images between libraries, and delete images from the libraries using the admin portal.

  • Playlist Creation and Management

    • The user should be able to select a template, add images from the libraries to the playlists and adjust the settings (ex. slide duration).

  • Assign Playlists to a Device

    • Devices will launch a browser in full-screen kiosk mode at startup and open the index.html file on the webserver that is stored in the folder under /devices that matches their hostname. For example: If the hostname of a device is "pisign1" and the server name is "signage.edu", the device should open https://signage.edu/devices/pisign1/index.html at launch.

    This index file is a simple redirect:

    
<meta http-equiv="refresh" content="0; URL=https://signage.cvad.unt.edu/playlist/playlist-name/hostname/index.html"/>
    The playlist is assigned to the device through the admin portal by replacing the string ''playlist-name'' in the index.html file.

Screensavers:

  • Simple screensavers for Mac and Windows will be created that points to a default playlist on the server. The default playlist should be configurable, with the ability to set the default playlist before it is distributed, but not edittable by the end user. The intent is that the user does not have to configure anything - just install and let the OS settings on time to wait before a screensaver begins handle the rest.

Digital Signage Player:

  • RaspberryPi: FullPageOS: Simple Raspberry Pi OS build that launches a web browser in kios mode and opens a specified URL at startup. Since this exists already, we will just need to include instructions for setting the devices up.

Content

The content will be provided through webpages hosted on the webserver. The templates should display the content such that it fits the screen without scrolling. The presentation of the content will be controlled entirelly by the html - the devices will just point at the pages.

  • Images: The images should be created with 16:10 aspect ratio.
  • Templates: The links to the starter versions of three templates we will use at the start of the project can be found be found at the bottom of this page. Some tweaking will need to occur: for exaple the images that do not have a 16:10 aspect ration will need to be adjusted to fit the screen, which may require a background be added (black).

Admin Console Features

Options: Device, Groups, Images, Playlists, Scheduler

  • Device
  • Add\Remove Device (just creates\deletes the folder with the hostanem and index file with the redirect).
  • Groups
  • Allow user to create or delete a group.
  • Allow user to add or remove devices from a group.
  • Allow user to assign a playlist to a group.
  • When a device is removed from a group, the hostname index.html should be updated so that it points to the default playlist.
  • Images
  • Create or delete image libraries.
  • Upload, download or delete Images from a library.
  • Playlists
  • Create, edit or delete playlists.
  • Add\Remove images from from a playlist.
  • Edit playlists (order the slides, set duration for each image, set transition type)
  • Scheduler
  • Create an assignable schedule for playlist changes: daily schedules, and by date.
  • User should be able to assign a scheudle to a device or group of devices
